# Microproyecto: Utilidades Numéricas en ARM64 Assembly para GitHub Classroom

## Contexto
En cursos de arquitectura de computadoras y sistemas embebidos, es clave que el estudiantado relacione la teoría del conjunto de instrucciones con una implementación funcional mínima. Esta actividad propone un microproyecto de bajo costo computacional, orientado a comprender el flujo de datos, llamadas al sistema y organización básica de un repositorio técnico en GitHub Classroom.

## Objetivo general
Diseñar, documentar e implementar un microproyecto con núcleo en ARM64 Assembly que procese una entrada numérica desde terminal y ejecute funciones básicas de validación y cálculo, empleando al menos una macro de Assembly y un plan de pruebas medible.

## Competencias
- Interpreta y aplica instrucciones básicas de ARM64 Assembly en un problema concreto.
- Documenta decisiones técnicas mediante artefactos de ingeniería claros y verificables.
- Diseña pruebas funcionales con criterios de éxito observables.
- Gestiona entregables técnicos en un flujo académico con GitHub Classroom.

## Instrucciones rápidas
1. Crear el repositorio asignado en GitHub Classroom.
2. Revisar `docs/propuesta.md` y validar alcance de 3 a 5 funcionalidades pequeñas.
3. Implementar el núcleo en ARM64 Assembly (máximo sugerido ~150 líneas funcionales).
4. Incluir al menos una macro funcional (por ejemplo, para impresión o salida controlada).
5. Ejecutar pruebas definidas en `docs/plan_de_pruebas.md`.
6. Entregar evidencias en commit final con documentación completa.

## Criterios de entrega
- Se entregan completos los 6 archivos solicitados.
- La propuesta, caso de uso, estructura y plan de pruebas son consistentes entre sí.
- El diseño contempla ARM64 Assembly como núcleo obligatorio.
- El plan de pruebas permite evaluación por competencias y resultados observables.
- La reflexión final reporta uso responsable de IA e integridad académica.

## Rúbrica
| Criterio | Excelente (100) | Satisfactorio (80) | Básico (70) | Insuficiente (<70) |
|---|---|---|---|---|
| Diseño técnico | Arquitectura clara, límites precisos y riesgos mitigados | Diseño funcional con ligeras omisiones | Diseño incompleto pero entendible | Sin coherencia técnica |
| Implementación ARM64 | Núcleo funcional con macro bien aplicada y código legible | Funciona con detalles menores por corregir | Funciona parcialmente | No cumple núcleo ARM64 |
| Pruebas y evidencia | Casos completos, medibles y reproducibles | Casos suficientes con evidencia parcial | Pocas pruebas o ambiguas | Sin pruebas verificables |
| Documentación | Redacción técnica clara y alineada a competencias | Documentación adecuada con pequeños vacíos | Documentación mínima | Documentación deficiente |
| Integridad académica | Declara uso de IA, valida manualmente y sustenta decisiones | Declara uso de IA con validación limitada | Declaración incompleta | Sin declaración |
